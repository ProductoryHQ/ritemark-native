If you fork this project, you are welcome to use all the source code under the MIT License. However, you **must**:

1. Choose a different product name
2. Replace all Ritemark logos and icons (the `branding/` directory assets)
3. Replace the bundle identifier (`ai.productory.ritemark`)
4. Remove any references that would confuse users into thinking your fork is the official Ritemark

This is the same model used by Mozilla Firefox (forks must rebrand — e.g., "Iceweasel"), VS Code (the open-source build is "Code - OSS", not "Visual Studio Code"), and many other open-source projects.
