# Ritemark - Brand Guidelines

**Version:** 1.0
**Date:** 2025-12-31

---

## Brand Personality

### Five Core Traits

| Trait | Definition | Example |
|-------|------------|---------|
| **Pragmatic** | Solutions over promises | "Markdown works for AI. DOCX doesn't. That's architecture, not opinion." |
| **Clear-Eyed** | See the shift, say it plainly | "By 2026, at least one Fortune 500 adopts markdown docs. Not because it's trendy—because consultants will force it." |
| **Developer-Native** | Technical precision, zero fluff | "VS Code fork with patches, not Electron bloat. Local-first means actually local." |
| **Confident but Humble** | Strong opinions, loosely held | "We think this is the future. If we're wrong, we built a decent editor. If we're right, you got here early." |
| **Utility-First** | Tool disappears, work happens | "Everything you need, nothing you don't." |

### What We Are vs. What We're NOT

| We Are | We're NOT |
|--------|-----------|
| Focused tool for clear use case | All-in-one productivity suite |
| AI-native from day one | AI bolted onto legacy software |
| Opinionated about format | Trying to please everyone |
| Local-first, privacy-respecting | Cloud service that works offline |
| For serious writers | Toy for casual note-taking |

**Sentence Structure:**
- Short sentences. Active voice. Say it straight.
- Use technical terms correctly, explain when needed

## Words to Avoid

| Never Say | Say Instead |
|-----------|-------------|
| Revolutionary | "Built for AI-native workflow" |
| Game-changing | "Changes how you work" |
| Seamless integration | "Terminal runs inside the app" |
| Empower | "Give you control" |
| Leverage | (delete it) |
| World-class | "Built on VS Code foundation" |
| Cutting-edge | "Uses TipTap editor, Claude integration" |
| Next-generation | "Built for dual-use documents" |

## Competitor Comparison Rules

**Don't:**
- "Better than Notion"
- "Kills Word"
- "Obsidian replacement"

**Do:**
- Feature table comparisons without hyperbole
- "Different positioning from X"
- Let features speak: "Ritemark has built-in terminal. Obsidian uses plugins."

---

## Tone Mistakes to Avoid

**Arrogance:**
- ❌ "Join the smart ones early."
- ✅ "We think markdown is the future. Here's why."

**False Humility:**
- ❌ "We're just trying to make something useful, hopefully people like it!"
- ✅ "Built by Jarmo Tuisk at Productory AI. Available now."

**Defensiveness:**
- ❌ "Unlike bloated cloud tools that spy on you..."
- ✅ "Local-first. Your files on your machine."

**Vagueness:**
- ❌ "Helps you write better and work smarter."
- ✅ "Write in rich text. Save as markdown. Run Claude Code in terminal."

---

## Brand Voice Checklist

Before publishing, verify:

- [ ] Specific? (Avoid vague claims)
- [ ] True? (Can we prove it?)
- [ ] Clear? (Developer understands immediately?)
- [ ] No hype words? (Revolutionary, game-changing)
- [ ] Active voice? (Not passive constructions)
- [ ] Short sentences?
- [ ] Would Jarmo approve? (No MBA jargon)

## Visual Identity Notes

**Inherited from Productory:**
- Indigo primary (#4338ca)
- Bold, technical, vibrant

**Ritemark Adaptations:**
- Softer, writing-focused
- More neutral tones (slate, white space)
- Terminal aesthetic (monospace accents)
- Less gradient, more clarity

**Typography:**
- Headlines: Space Grotesk
- Body: Sofia Sans or JetBrains Mono for terminal feel
- Emphasis on readability

---

*The non-negotiables: No hype. Technical precision. Respect the audience. Let the product speak.*
