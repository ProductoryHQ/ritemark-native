## Executive Summary

RiteMark Native enters a $3.44B markdown editor market at a unique inflection point. The convergence of three trends creates a once-in-a-decade opportunity:

1. **AI is table stakes** — but all implementations require cloud APIs
2. **Privacy concerns accelerating** — 72% prefer on-premise solutions
3. **Subscription fatigue is real** — one-time pricing wins hearts

**Our position:** The only tool combining WYSIWYG Markdown + built-in AI terminal + local-first architecture.

No competitor offers this. That's our wedge.

## Strategic Positioning

### The Thesis

**"Documents must work for humans AND AI."**

This is the shift. DOCX/PDF fail this test. Markdown passes. We built the editor for this future.

### One-Liner Options (by audience)

| Audience | One-Liner |
|----------|-----------|
| Developers | "Claude Code, but for writing" |
| Knowledge Workers | "Stop copying between ChatGPT and Word" |
| Markdown Enthusiasts | "Your markdown editor, upgraded for the AI era" |
| General | "The text editor that speaks AI fluently" |

### Positioning Statement

> For **knowledge workers and Claude Code users** who need to **write documents with AI assistance**, RiteMark Native is a **desktop markdown editor** that **combines WYSIWYG editing with built-in terminal for AI agents**. Unlike Word (AI bolted on) or Notion (cloud-only), we're **built from the ground up for dual-use documents**.

## Message Architecture

### Core Narrative

The world is splitting into two eras:
- **Before:** Documents for humans (Word, PDF, Docs)
- **After:** Documents for humans AND AI (Markdown)

RiteMark Native is built for the "After."

### Three Pillars

| Pillar | Message | Proof Point |
|--------|---------|-------------|
| **Dual-Use Format** | Markdown works for humans AND AI | "DOCX loses half the info. Markdown keeps structure." |
| **AI-Native Workflow** | Claude Code built-in, not bolted on | "Terminal runs inside the app. Same window." |
| **Local-First Control** | Your files, your machine | "Works offline. No cloud required." |

### Objection Handling

| Objection | Response |
|-----------|----------|
| "I use Notion/Docs" | Cloud-only. Do you trust them with private notes? |
| "Why Markdown?" | Only format AI reads/writes fluently. Try giving Claude a DOCX. |
| "I can use VS Code" | You could. But it's for code. We stripped it for writing. |
| "What about Obsidian?" | Great tool. We add native AI terminal + WYSIWYG. |
| "Mac only?" | For now. Windows/Linux in roadmap. |

### Personality

- **Confident** — We know where the future is going
- **Practical** — No hype, just useful tools
- **Technical but Accessible** — Devs trust us, non-devs understand
- **Opinionated** — Markdown > DOCX. Local > Cloud.

### Voice Examples

**DO:**
> "Markdown is the only format that works for both you and your AI. That's not opinion—it's architecture."

**DON'T:**
> "RiteMark is an amazing revolutionary tool that will change how you write forever!!!"

## CMO Notes & Decisions

### What I Like About This Strategy

1. **Clear wedge** — "Local-first + AI terminal" is genuinely unique
2. **Audience focus** — Starting with Claude Code users (high comprehension, low education needed)
3. **Thesis-driven** — "Dual-use documents" is defensible, timely, and differentiating
4. **Organic-first** — $0 budget forces discipline, validates product-market fit

### What I'm Concerned About

1. **Mac-only** — Limits TAM significantly. Windows should be Q2 priority.
2. **Claude dependency** — Naming ties us to Anthropic. What if they pivot?
3. **"AI" positioning fatigue** — Market flooded with "AI writing tools." Need sharper differentiation.
4. **Enterprise path unclear** — How do we monetize? Free forever isn't a business.

### Decisions Made

1. **Primary tagline:** "Claude Code, but for writing" (developer-first)
2. **Launch channel:** ProductHunt first, then HN
3. **Pricing:** Free for now. Evaluate monetization after 1,000 users.
4. **Windows priority:** Move up roadmap if Phase 1 shows demand signals.

### Open Questions

1. Should we rebrand away from "Claude Code" reference? (Anthropic risk)
2. Is "RiteMark" memorable enough? Consider alternatives?
3. What's the monetization model? (Freemium? Pro tier? Donations?)
4. Should we open source fully for credibility?
