## Ritemark v1.5.2

**Status:** Open release  
**Type:** Minor release

## Highlights

Ritemark v1.5.2 brings Codex into Flows. While a dedicated **Claude** node already existed, there was no equivalent agent node for people who prefer OpenAI models or want to use their ChatGPT subscription for autonomous coding tasks. Now you can add a dedicated **Codex** node and use it the same way as Claude inside a visual workflow.

## What's New

### Codex Flow Node

- New **Codex** node in the Flow Editor AI section
- Drag the node onto the canvas and connect it the same way as **Claude Code**
- Configure prompt, model, and timeout directly from the node properties panel
- Flow execution now routes Codex nodes through the Codex CLI app-server

### Open Model Choice

Ritemark is intended to be an open platform, not a single-model product.

- Choose between Anthropic and OpenAI for agent workflows
- Use Claude and Codex side by side in the same product
- Build flows around the model ecosystem that fits your work best today
- Keep the door open for future additions such as Gemini

### Flow Editor Hardening

- Fixed Codex node configuration crash that could blank the flow editor after drop/select
- Added Codex-specific execution progress handling in the flow results panel
- Added regression coverage for flow node type mappings and Codex runtime wiring
