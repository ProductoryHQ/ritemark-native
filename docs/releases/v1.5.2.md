## Ritemark v1.5.2

**Status:** Open release  
**Type:** Minor release

## Highlights

Ritemark v1.5.2 brings two major workflow upgrades into Flows and adds a lightweight feedback loop inside the editor.

First, Flows now include a dedicated **Codex** node. While a dedicated **Claude** node already existed, there was no equivalent agent node for people who prefer OpenAI models or want to use their ChatGPT subscription for autonomous coding tasks. Now you can add a dedicated **Codex** node and use it the same way as Claude inside a visual workflow.

Second, Flows can now run automatically on a schedule while Ritemark is open for that workspace. This makes recurring automations possible without external cron setup.

Third, Ritemark now includes a quick in-editor **Reactions** action. You can send fast sentiment feedback directly from the editor toolbar, and the app can now record anonymous product usage events through a dedicated app analytics project.

## What's New

### Codex Flow Node

- New **Codex** node in the Flow Editor AI section
- Drag the node onto the canvas and connect it the same way as **Claude Code**
- Configure prompt, model, and timeout directly from the node properties panel
- Flow execution now routes Codex nodes through the Codex CLI app-server

### Scheduled Flow Runs

- Add a flow-level schedule directly from the Flow Editor
- Run eligible flows automatically while Ritemark is open
- Supported recurrence types include daily, weekdays, weekly, hourly, and short minute intervals
- View next run timing and last scheduled run status from the flow schedule panel
- Pause or remove the schedule without removing the flow itself

### Open Model Choice

Ritemark is intended to be an open platform, not a single-model product.

- Choose between Anthropic and OpenAI for agent workflows
- Use Claude and Codex side by side in the same product
- Build flows around the model ecosystem that fits your work best today
- Keep the door open for future additions such as Gemini

### Editor Reactions

- New **Send Feedback** action in the editor toolbar across files
- Submit a quick reaction and optional free-text note without leaving your work
- Anonymous app analytics now flow into a dedicated PostHog app project instead of the web analytics project
- First-launch analytics notice now links directly to Privacy Policy and Terms of Use

### Flow Editor Hardening

- Fixed Codex node configuration crash that could blank the flow editor after drop/select
- Added Codex-specific execution progress handling in the flow results panel
- Added regression coverage for flow node type mappings and Codex runtime wiring
- Added schedule runtime dedupe and overlap protection for long-running scheduled flows
- Fixed analytics startup sequencing so opt-out can apply before the first app session event is sent
