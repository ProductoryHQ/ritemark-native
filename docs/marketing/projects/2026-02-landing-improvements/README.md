Add concrete use cases and before/after workflow comparison to convert visitors who need to see real-world value before downloading.

## Problem

Current landing page:

-   **Use cases are vague** - "Professionals who write a lot" doesn't show what RiteMark actually helps with
    
-   **No before/after comparison** - Visitors don't see how their workflow improves
    

Visitors who are intrigued by the hero need more convincing before they scroll to download.

* * *

## Key Differentiator (Jarmo Input)

**AI agents work WITH YOU on the sidebar - not copy-paste between browser and editor.**

Tools like Claude Code, Gemini CLI, and Codex can see and edit your files directly. No more:

-   Copying text out of your document
    
-   Pasting into a browser chat
    
-   Waiting for a response
    
-   Copying the result back
    
-   Reformatting everything

## Before/After Workflow Comparison

### BEFORE: ChatGPT Workflow (6 steps)

1.  Open your document in one app
    
2.  Copy text you want AI to work with
    
3.  Switch to ChatGPT in browser
    
4.  Paste text and explain what you need
    
5.  Copy AI's response
    
6.  Paste back into document and reformat
    

**Pain points:** Context switching, manual copy-paste, reformatting, losing document structure

### AFTER: RiteMark Workflow (2 steps)

1.  Open document in RiteMark
    
2.  AI agent sees file and edits directly
    

**Key message:** AI works alongside you in the sidebar. It sees your file. It edits in place. No copy-paste.
